# Smart Study Calendar — StudyPlan

**Intelligent User Interfaces · Summer 2026 · Group 7**
Muhammad Shahzaib · Chengzhao Dai · Luna Luo Yunru · Kasi Nathan Murugan

Smart Study Calendar is an adaptive study planner that learns **when you study most effectively**, generates optimized weekly schedules, and explains every scheduling decision through a Bayesian learning model, schedule optimization engine, and Gemini-powered AI explanations.

---

## Features

* Weekly study planner with drag-and-drop scheduling
* Adaptive Bayesian focus model that learns from task completion and skip feedback
* Schedule optimization based on deadlines, difficulty, priority, workload balance, spacing, and user preferences
* Explainable scheduling decisions with score breakdowns and natural-language explanations
* Analytics dashboard with focus-profile heatmap
* Gemini-powered AI explanations with automatic fallback to the engine explanation
* Persistent user learning profile stored in `user_model.json`
* Local JavaScript scheduling fallback when the backend is unavailable

## Architecture

### Frontend

**React 19 + Vite + Tailwind CSS** (`src/`)

Responsible for:

* Task management
* Weekly calendar view
* Drag-and-drop rescheduling
* Analytics heatmap
* User preferences and settings
* Gemini explanation interface

If the backend is unavailable, the frontend falls back to a lightweight local JavaScript scheduling heuristic.

### Backend

**Python FastAPI** (`backend/`)

#### `engine.py`

Contains the core intelligence of the application.

##### FocusModel

A Beta-Bernoulli Bayesian learning model that:

* Learns completion probabilities for each day × time-slot
* Updates continuously from Done/Skip feedback
* Estimates when the user studies most effectively
* Persists learned behaviour across sessions

##### ScheduleOptimizer

A utility-based scheduling engine that considers:

* Learned completion probability
* Deadline pressure
* Task difficulty
* Task priority
* User focus preferences
* Workload balance
* Spacing and retention effects

The optimizer uses:

* Greedy construction
* Stochastic local search

to generate an optimized weekly study schedule.

Each placement includes a machine-generated explanation and score breakdown.

#### `main.py`

Provides the following API endpoints:

* `POST /api/schedule`
* `POST /api/feedback`
* `GET /api/model`
* `POST /api/model/reset`
* `GET /api/health`
* `POST /api/ai/explain`

### Gemini Integration

Gemini is used as an **explanation layer**, not as the scheduling engine itself.

Workflow:

```txt
Scheduler
   ↓
Explanation Engine
   ↓
Gemini
   ↓
User-Friendly Explanation
```

The scheduler first generates objective reasoning based on focus probabilities, deadlines, preferences, and workload constraints.

Gemini then converts this reasoning into concise, natural-language explanations that are easier for users to understand.

If Gemini is unavailable, the system automatically falls back to the built-in explanation generated by the optimization engine.

### Persistence

User learning data is stored in:

```txt
backend/user_model.json
```

A pre-seeded demo profile containing 83 feedback events is included and can be reset from the Settings page.

## Setup

### 1. Clone the Repository

```bash
git clone <repository-url>
cd StudyPlan
```

### 2. Create a Python Virtual Environment (Recommended)

#### Windows (Command Prompt)

```bash
venv\Scripts\activate
```

#### Windows (PowerShell)

```powershell
.\venv\Scripts\Activate.ps1
```

#### macOS / Linux

```bash
cd backend
python -m venv venv
source venv/bin/activate
```

### 3. Install Backend Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Gemini

Create:

```txt
backend/.env
```

Add:

```env
GEMINI_API_KEY=your_api_key_here
```

If Gemini is unavailable, the application automatically falls back to the scheduler's built-in explanation engine.

## Demo Script (~2 Minutes)

### 1. Create Tasks

* Navigate to **Tasks**
* Add several tasks with different deadlines, priorities, and difficulty levels
* Click **Generate Weekly Plan**

The optimizer automatically places study blocks into suitable time slots.

### 2. Explain a Scheduling Decision

* Open any scheduled study block
* Click **AI Explain**

The scheduling engine generates evidence-based reasoning and Gemini converts it into a concise, user-friendly explanation.

### 3. Demonstrate Learning

* Mark several tasks as completed
* Skip a few tasks in a specific time slot
* Regenerate the schedule

The Bayesian model updates completion probabilities and adapts future scheduling decisions accordingly.

### 4. View Analytics

* Open **Analytics**
* Observe the focus-profile heatmap

The heatmap visualizes learned completion probabilities across the week.

### 5. Change Preferences

* Open **Settings**
* Change the preferred focus window (Morning, Afternoon, or Evening)
* Regenerate the schedule

Hard tasks migrate toward the selected focus window.
